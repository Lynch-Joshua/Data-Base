# Data-Base
Proyecto de bases de datos relacionales sql para proceso de Matricula

## Descripcion
Nombre del proyecto
SchoolEnroll System, en español Sistema de inscripción escolar.
## Objetivos del Proyecto
El objetivo principal del proyecto es desarrollar un sistema en SQL que permita administrar y organizar el proceso de matrícula escolar de manera rápida, segura y eficiente.
Entre los objetivos específicos se encuentran registrar estudiantes nuevos y regulares, organizar la información por grado, bachiller y turno, controlar los cupos disponibles y evitar errores o duplicación de datos. Además, el sistema permitirá generar reportes automáticos y estadísticas relacionadas con las matrículas escolares.
## ¿Por qué se hace? ¿Para qué? ¿Qué va a cubrir?
Este proyecto se realiza con el propósito de mejorar el proceso de matrícula del colegio, ya que muchas veces los registros manuales generan retrasos, pérdida de información y desorganización.
El sistema servirá para agilizar el registro de estudiantes, facilitar la consulta de información y mantener una mejor organización de los datos académicos. También permitirá llevar un control de los cupos disponibles y generar estadísticas útiles para la administración escolar.
El proyecto cubrirá el registro de estudiantes, acudientes, grados, bachilleres, turnos y el proceso completo de matrícula.
## ¿Dónde se realizará el proyecto?
El proyecto utilizará diferentes herramientas tecnológicas para el diseño y funcionamiento del sistema de matrícula escolar.
La base de datos será desarrollada en SQL para almacenar y organizar la información de los estudiantes y matrículas. Además, se emplearán herramientas de diseño para crear el bosquejo visual de la interfaz del sistema y mostrar cómo funcionaría el entorno gráfico del proyecto.
También se utilizarán programas de administración de bases de datos como MySQL Workbench o SQL Server Management Studio para gestionar la información del sistema.


## Organización del Proyecto
El proyecto será desarrollado por cinco estudiantes, donde cada integrante tendrá una responsabilidad específica para lograr una mejor organización del trabajo.
Un integrante se encargará del diseño de la interfaz del sistema. Otro será responsable de la creación de la base de datos en SQL. El tercer integrante desarrollará los procesos automatizados del sistema. El cuarto estudiante trabajará en los reportes y estadísticas. Finalmente, el quinto integrante se encargará de la documentación y exposición del proyecto.
## Entorno gráfico
El sistema contará con una interfaz sencilla y organizada para facilitar su uso. Tendrá una pantalla principal con un menú lateral que permitirá acceder a las diferentes funciones del sistema.
Entre las opciones estarán el registro de estudiantes, registro de acudientes, proceso de matrícula, consulta de estudiantes y generación de reportes. También se incluirán botones para registrar, editar, eliminar y buscar información rápidamente.
## Idea de la Base de Datos
La base de datos estará diseñada en SQL y contará con varias tablas relacionadas entre sí para mantener la información organizada.
La tabla de estudiantes almacenará datos como nombre, apellido, cédula, grado y bachiller. La tabla de acudientes guardará información del responsible del estudiante, incluyendo teléfono y dirección. También existirá una tabla de matrícula donde se registrará la fecha de matrícula, turno y estado del estudiante.
Estas tablas estarán conectadas mediante claves primarias y foráneas para asegurar un mejor control de la información.
## Automatización del sistema
El proyecto incluirá procesos automatizados dentro de la base de datos utilizando herramientas de SQL como procedimientos almacenados y consultas automáticas. Estas funciones permitirán registrar estudiantes, actualizar información y organizar los datos de matrícula de manera más rápida y eficiente.
Además, el sistema podrá generar reportes y estadísticas sobre la cantidad de estudiantes matriculados por grado, bachiller o turno, facilitando el control y administración de la información escolar.
